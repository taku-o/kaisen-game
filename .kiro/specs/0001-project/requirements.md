# 海戦ゲーム 要件定義書

## 1. 概要

### 1.1 プロジェクトの目的

古典的な海戦ゲーム（Battleship）をWebアプリケーションとして実装する。2人のプレイヤーが1台の端末で交互にプレイするローカル対戦を提供する。

### 1.2 スコープ

本プロジェクトは以下を対象とする:

- PCデスクトップブラウザ向けWebアプリケーション
- 1台の端末で2人が交互にプレイするローカル対戦
- 古典的な海戦ゲームの基本ルールに準拠した対戦機能

---

## 2. 機能要件

### 要件1: タイトル画面

#### 概要

ゲーム起動時に表示される最初の画面。ゲームの開始操作を提供する。

#### 機能詳細

- タイトル画面にゲームタイトルを表示する
- タイトル画面にゲーム開始ボタンを表示する

#### 受け入れ基準

| No | 種別 | 基準 |
|----|------|------|
| 1-1 | Ubiquitous | システムはタイトル画面にゲームタイトルを表示する |
| 1-2 | Ubiquitous | システムはタイトル画面にゲーム開始ボタンを表示する |
| 1-3 | Event-Driven | ユーザーがゲーム開始ボタンをクリックした時、システムは配置画面へ遷移する |

---

### 要件2: 艦配置機能

#### 概要

各プレイヤーが自分の艦隊を10×10グリッド上に配置する機能。

#### 機能詳細

- 10×10マスのグリッドを表示する（縦軸: A〜J、横軸: 1〜10）
- 5隻の艦を配置する
  - 空母（Aircraft Carrier）: 5マス
  - 戦艦（Battleship）: 4マス
  - 巡洋艦（Cruiser）: 3マス
  - 潜水艦（Submarine）: 3マス
  - 駆逐艦（Destroyer）: 2マス
- 艦の向き（水平/垂直）を切り替える機能を提供する
- 現在のプレイヤー（Player 1 / Player 2）を表示する
- 配置を確定するボタンを提供する

#### 配置ルール

- 艦は水平または垂直に配置する（斜めは不可）
- 艦はグリッドの境界線を超えてはならない
- 艦同士が同じマスを共有することはできない

#### 受け入れ基準

| No | 種別 | 基準 |
|----|------|------|
| 2-1 | Ubiquitous | システムは10×10マスのグリッド（縦軸A〜J、横軸1〜10）を表示する |
| 2-2 | Ubiquitous | システムは配置する艦の一覧（空母5マス、戦艦4マス、巡洋艦3マス、潜水艦3マス、駆逐艦2マス）を表示する |
| 2-3 | Ubiquitous | システムは現在のプレイヤー（Player 1 / Player 2）を表示する |
| 2-4 | Ubiquitous | システムは艦の向き切り替え機能（水平/垂直）を提供する |
| 2-5 | Ubiquitous | システムは配置確定ボタンを表示する |
| 2-6 | Event-Driven | ユーザーがグリッド上のマスをクリックした時、システムは選択中の艦をそのマスを起点として配置する |
| 2-7 | Event-Driven | ユーザーが艦の向き切り替え操作をした時、システムは配置方向を水平と垂直で切り替える |
| 2-8 | Unwanted Behavior | 艦がグリッドの境界線を超える配置をしようとした場合、システムはその配置を許可しない |
| 2-9 | Unwanted Behavior | 艦が既に配置された艦と重なる配置をしようとした場合、システムはその配置を許可しない |
| 2-10 | State-Driven | 全ての艦（5隻）が配置された状態の時、システムは配置確定ボタンを有効にする |
| 2-11 | State-Driven | 艦が未配置の状態の時、システムは配置確定ボタンを無効にする |
| 2-12 | Event-Driven | Player 1が配置確定ボタンをクリックした時、システムはPlayer 2の配置画面へ遷移する |
| 2-13 | Event-Driven | Player 2が配置確定ボタンをクリックした時、システムは先攻プレイヤーを決定し対戦画面へ遷移する |

---

### 要件3: 対戦機能

#### 概要

プレイヤーが交互に相手の艦を攻撃する機能。

#### 機能詳細

- 攻撃記録グリッド（相手への攻撃結果を表示）を表示する
- 自陣グリッド（自分の艦と被弾状況を表示）を表示する
- 現在のターン（どちらのプレイヤーか）を表示する
- 攻撃結果メッセージ（命中 / ミス / 撃沈）を表示する
- 残存艦の状況を表示する

#### 攻撃ルール

- プレイヤーは攻撃記録グリッド上のマスをクリックして攻撃座標を選択する
- 既に攻撃済みのマスは再度攻撃できない

#### 受け入れ基準

| No | 種別 | 基準 |
|----|------|------|
| 3-1 | Ubiquitous | システムは攻撃記録グリッド（相手への攻撃結果を表示するグリッド）を表示する |
| 3-2 | Ubiquitous | システムは自陣グリッド（自分の艦と被弾状況を表示するグリッド）を表示する |
| 3-3 | Ubiquitous | システムは現在のターン（Player 1 / Player 2）を表示する |
| 3-4 | Ubiquitous | システムは両プレイヤーの残存艦の状況を表示する |
| 3-5 | Event-Driven | ユーザーが攻撃記録グリッド上の未攻撃マスをクリックした時、システムはそのマスへ攻撃を実行する |
| 3-6 | Event-Driven | 攻撃がミスだった時、システムはミスを示す表示を攻撃記録グリッドに表示する |
| 3-7 | Event-Driven | 攻撃が命中だった時、システムは命中を示す表示を攻撃記録グリッドに表示する |
| 3-8 | Event-Driven | 攻撃結果が判明した時、システムは攻撃結果メッセージ（命中 / ミス）を表示する |
| 3-9 | Event-Driven | 攻撃により艦が撃沈した時、システムは撃沈メッセージを表示する |
| 3-10 | Unwanted Behavior | 既に攻撃済みのマスをクリックした場合、システムは攻撃を実行しない |
| 3-11 | Event-Driven | 攻撃が終了し勝敗が決していない時、システムはターンを相手プレイヤーに交代する |
| 3-12 | Event-Driven | ターンが交代した時、システムは相手プレイヤーの配置が見えないように画面を切り替える |

---

### 要件4: 勝敗判定機能

#### 概要

相手の艦隊を全て撃沈した時点で勝利を判定する機能。

#### 機能詳細

- 艦の全マスが命中した場合、その艦は撃沈となる
- 相手の全ての艦（5隻）を撃沈した場合、勝利となる

#### 受け入れ基準

| No | 種別 | 基準 |
|----|------|------|
| 4-1 | Event-Driven | 艦の全マスが命中した時、システムはその艦を撃沈状態とする |
| 4-2 | Event-Driven | 相手の全ての艦（5隻）が撃沈した時、システムは攻撃したプレイヤーを勝者と判定する |
| 4-3 | Event-Driven | 勝者が決定した時、システムは結果画面へ遷移する |

---

### 要件5: 結果画面

#### 概要

勝敗が決定した後に表示される画面。勝者を表示し、ゲームを再開するための操作を提供する。

#### 機能詳細

- 勝者（Player 1 / Player 2）を表示する
- タイトル画面に戻るボタンを表示する

#### 受け入れ基準

| No | 種別 | 基準 |
|----|------|------|
| 5-1 | Ubiquitous | システムは勝者（Player 1 / Player 2）を表示する |
| 5-2 | Ubiquitous | システムはタイトルに戻るボタンを表示する |
| 5-3 | Event-Driven | ユーザーがタイトルに戻るボタンをクリックした時、システムはタイトル画面へ遷移する |

---

### 要件6: 画面遷移

#### 概要

ゲームの進行に応じて画面を切り替える機能。

#### 画面構成

1. タイトル画面
2. 配置画面
3. 対戦画面
4. 結果画面

#### 受け入れ基準

| No | 種別 | 基準 |
|----|------|------|
| 6-1 | Ubiquitous | システムはタイトル画面、配置画面、対戦画面、結果画面の4画面で構成される |
| 6-2 | Event-Driven | ゲーム開始ボタンがクリックされた時、システムはタイトル画面から配置画面へ遷移する |
| 6-3 | Event-Driven | Player 1が配置を確定した時、システムはPlayer 2の配置画面へ遷移する |
| 6-4 | Event-Driven | Player 2が配置を確定した時、システムは対戦画面へ遷移する |
| 6-5 | Event-Driven | 勝者が決定した時、システムは対戦画面から結果画面へ遷移する |
| 6-6 | Event-Driven | タイトルに戻るボタンがクリックされた時、システムは結果画面からタイトル画面へ遷移する |

---

## 3. 非機能要件

### 3.1 対応ブラウザ

- PCデスクトップブラウザを対象とする

### 3.2 技術制約

- クライアントサイドで完結する（サーバーサイドは不要）
- SPA（Single Page Application）として実装する
- TypeScriptを使用する

### 3.3 対戦方式

- ローカル対戦: 1台の端末で2人のプレイヤーが交互にプレイする
- ターン交代時には画面を切り替え、相手の配置が見えないようにする

---

## 4. スコープ外

以下の機能は本プロジェクトでは実装しない:

- オンライン対戦
- AI対戦
- モバイル対応
- Salvoモード（複数回攻撃）
- 艦の移動ルール
- 連続攻撃ルール（命中時の追加攻撃）
- 隣接禁止ルール
- 簡易版（5×5グリッド等）
- サウンド・BGM
- アニメーション効果
- ゲームの保存・再開機能
- 戦績の記録
