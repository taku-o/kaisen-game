# 海戦ゲーム 企画書

## 1. ゲーム概要

本プロジェクトは、古典的な海戦ゲーム（Battleship）をWebアプリとして再現するものです。

2人のプレイヤーが10×10のグリッド上に艦隊を秘密裏に配置し、交互に相手の艦の位置を推理しながら攻撃を行います。相手の艦隊を全て撃沈したプレイヤーが勝利となります。

## 2. 対象プラットフォーム

- **プラットフォーム**: Webブラウザ
- **対応デバイス**: PC（デスクトップブラウザ）

## 3. ゲームルール

### 3.1 グリッド

- サイズ: 10×10マス
- 縦軸: A〜J（アルファベット）
- 横軸: 1〜10（数字）

### 3.2 艦の種類

| 艦種 | マス数 |
|------|--------|
| 空母（Aircraft Carrier） | 5 |
| 戦艦（Battleship） | 4 |
| 巡洋艦（Cruiser） | 3 |
| 潜水艦（Submarine） | 3 |
| 駆逐艦（Destroyer） | 2 |

合計: 5隻、17マス

### 3.3 配置ルール

1. 艦は水平または垂直に配置する（斜めは不可）
2. 艦はグリッドの境界線を超えてはならない
3. 艦同士が同じマスを共有することはできない
4. ゲーム開始後、艦を移動することはできない

### 3.4 ゲームの流れ

1. **配置フェーズ**: 両プレイヤーが自陣グリッドに5隻の艦を配置する
2. **先攻決定**: 先攻プレイヤーを決定する
3. **攻撃フェーズ**: プレイヤーは交互に以下を行う
   - 攻撃するマスの座標を選択する
   - 結果が表示される（命中 / ミス）
   - 艦の全マスが命中した場合、撃沈が通知される
4. **ターン交代**: 攻撃が完了したら相手のターンに移る

### 3.5 勝利条件

相手の艦隊（5隻、17マス）を全て撃沈したプレイヤーが勝利

## 4. 画面構成

### 4.1 タイトル画面

- ゲームタイトルの表示
- ゲーム開始ボタン

### 4.2 配置画面

- 自陣グリッド（10×10）の表示
- 配置する艦の一覧
- 艦の向き切り替え機能（水平/垂直）
- 配置確定ボタン
- 現在のプレイヤー表示（Player 1 / Player 2）

### 4.3 対戦画面

- 攻撃記録グリッド（相手への攻撃結果を表示）
- 自陣グリッド（自分の艦と被弾状況を表示）
- 現在のターン表示
- 攻撃結果メッセージ（命中 / ミス / 撃沈）
- 残存艦の表示

### 4.4 結果画面

- 勝者の表示
- タイトルに戻るボタン

## 5. 対戦方式

- **ローカル対戦**: 1台の端末で2人のプレイヤーが交互にプレイする
- ターン交代時には画面を切り替え、相手の配置が見えないようにする

## 6. 基本機能一覧

| 機能 | 説明 |
|------|------|
| 艦配置機能 | 5隻の艦をグリッド上に配置する |
| 艦の向き切り替え | 配置時に水平/垂直を切り替える |
| 配置バリデーション | 配置ルールに違反していないか確認する |
| 攻撃機能 | 座標を選択して攻撃を実行する |
| 命中判定 | 攻撃が命中したかミスかを判定する |
| 撃沈判定 | 艦の全マスが命中したかを判定する |
| 勝敗判定 | 全艦撃沈で勝利を判定する |
| ターン管理 | プレイヤーのターンを交互に切り替える |
| 画面遷移 | 各画面間の遷移を制御する |
